1//! Numerical methods for solving differential equations
2//!
3//! This module contains concrete implementations of the [`Solver`](crate::solver::Solver) trait.
4//!
5//! # Architecture
6//!
7//! The separation between abstract solver interface (`solver::traits`) and concrete
8//! implementations (`solver::methods`) follows the Open-Closed Principle:
9//! - **Open** for extension: Add new methods without modifying existing code
10//! - **Closed** for modification: The `Solver` trait is stable and never changes
11//!
12//! # Available Methods
13//!
14//! ## Explicit Time-Stepping Methods
15//!
16//! These methods are suitable for non-stiff ordinary differential equations (ODEs)
17//! where the right-hand side function can be evaluated explicitly.
18//!
19//! - **[`EulerSolver`]**: Forward Euler method
20//!   - Order: First-order O(dt)
21//!   - Cost: 1 function evaluation per step
22//!   - Use: Prototyping, educational purposes, non-stiff problems with relaxed accuracy
23//!
24//! - **[`RK4Solver`]**: Classical fourth-order Runge-Kutta
25//!   - Order: Fourth-order O(dt⁴)
26//!   - Cost: 4 function evaluations per step
27//!   - Use: **Production simulations**, non-stiff to moderately stiff problems
28//!
29//! # Future Methods (Planned)
30//!
31//! ## Adaptive Methods (v0.2.0+)
32//! - **RK45**: Runge-Kutta-Fehlberg with adaptive step size control
33//! - **Dormand-Prince**: Higher-order adaptive method
34//!
35//! ## Implicit Methods (v0.3.0+)
36//! - **BDF**: Backward Differentiation Formulas for stiff problems
37//! - **Rosenbrock**: Semi-implicit methods
38//!
39//! ## Iterative Methods (v0.4.0+)
40//! - **Newton-Raphson**: For steady-state problems
41//! - **GMRES**: For large linear systems
42//!
43//! # Example
44//!
45//! ```rust
46//! use chrom_rs::solver::{EulerSolver, RK4Solver};
47//! use chrom_rs::solver::{Scenario, DomainBoundaries, Solver, SolverConfiguration};
48//! use chrom_rs::physics::{ PhysicalModel, PhysicalState, PhysicalQuantity, PhysicalData };
49//! use nalgebra::DVector;
50//!
51//! struct MyModel;
52//!
53//! impl PhysicalModel for MyModel {
54//!      fn points(&self) -> usize { 1 }
55//!      fn compute_physics(&self, state: &PhysicalState) -> PhysicalState { state.clone() }
56//!      fn setup_initial_state(&self) -> PhysicalState {
57//!          PhysicalState::new(PhysicalQuantity::Concentration, PhysicalData::Vector(DVector::from_vec(vec![1.0])))
58//!      }
59//!      fn name(&self) -> &str { "MyModel" }
60//!  }
61//!
62//! fn main() -> Result<(), String> {
63//!
64//!     let model = Box::new(MyModel);
65//!     let boundaries = DomainBoundaries::temporal(model.setup_initial_state());
66//!     let scenario = Scenario::new(model, boundaries);
67//!
68//!
69//!     // Using Euler for production
70//!     let euler = EulerSolver::new();
71//!     let configuration = SolverConfiguration::time_evolution(600.0, 10000);
72//!     let result = euler.solve(&scenario, &configuration)?;
73//!
74//!     // Using Runge-Kutta for production
75//!     let rk4 = RK4Solver::new();
76//!     let configuration = SolverConfiguration::time_evolution(600.0, 10000);
77//!     let result = euler.solve(&scenario, &configuration)?;
78//!
79//!     Ok(())
80//! }
81//!
82//! ```
83//!
84//! # Design Philosophy
85//!
86//! Each solver is:
87//! - **Self-contained**: No shared mutable state
88//! - **Stateless**: Can be reused for multiple simulations
89//! - **Well-tested**: 30+ tests per solver with ~95% coverage
90//! - **Documented**: Complete rustdoc with mathematical background
91//!
92//! # Performance Considerations
93//!
94//! All solvers benefit from:
95//! - **Rayon parallelization** (feature `parallel`) for large spatial grids
96//! - **Configurable threshold** via `set_parallel_threshold()`
97//! - **Efficient nalgebra** operations for matrix arithmetic
98//!
99//! See individual solver documentation for specific performance characteristics.
